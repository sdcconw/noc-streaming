# NOC Streaming 要件定義書
作成日: 2026-03-07
更新日: 2026-03-11

## 1. 目的

NOC 監視用の Web 画面やターミナル画面をキャプチャし、RTMP または SRT で継続配信する。
運用者は WebUI からジョブの作成、配信開始、停止、再起動、状態確認を実施できること。

## 2. システム構成要件

本システムは以下の構成で動作すること。

- WebUI: Bootstrap ベースの管理画面
- アプリケーションサーバー: Node.js 24 / TypeScript / Express
- API 契約管理: OpenAPI 3.1 / Swagger UI
- データベース: MySQL 8.4
- ORM: Prisma
- 配信ワーカー: Xvfb / Chromium / FFmpeg / x11vnc / xterm
- ブラウザ VNC 閲覧: noVNC
- コンテナ管理: Docker Compose

## 3. 提供機能要件

### 3.1 ジョブ管理

運用者は WebUI および API から以下を実施できること。

- ジョブ一覧表示
- ジョブ作成
- ジョブ編集
- ジョブ削除
- ジョブ開始
- ジョブ停止
- ジョブ再起動
- ジョブログ閲覧
- ジョブ状態確認
- noVNC 起動

### 3.2 管理機能

運用者は以下を実施できること。

- ローカルユーザー管理
- LDAP 設定管理
- LDAP 接続確認
- 監査ログ閲覧
- ダークモード切替

## 4. ジョブ設定要件

ジョブは以下の設定値を保持できること。

- ジョブ名
- 映像幅
- 映像高さ
- FPS
- ビットレート
- コーデック
- プリセット
- 出力プロトコル
- 出力 URL
- Stream Key（任意、空文字可）
- 入力ソース種別
- 画面切替間隔
- 最大再接続回数
- 再接続間隔
- オーバーレイ有効/無効
- オーバーレイメッセージ
- オーバーレイ文字サイズ
- オーバーレイ表示位置
- テストパターン種別
- テストパラメータ
- SSH ターミナル背景色
- SSH ターミナル文字色
- 表示 URL 一覧

## 5. 入力ソース要件

### 5.1 Browser

- Chromium で対象 URL を表示すること
- Xvfb 上のブラウザ画面を FFmpeg で配信すること
- ブラウザ画面ではマウスカーソルを表示しないこと
- 可能な限り表示領域を有効活用し、不要な黒帯を抑制すること

### 5.2 Test Pattern

- FFmpeg の `lavfi` によりテストパターンを生成できること
- テストパターンの種類とパラメータをジョブごとに指定できること

### 5.3 SSH Terminal

- Xvfb 上で xterm を起動できること
- 起動時は空のログインシェルを表示すること
- SSH 接続先や認証情報は自動投入せず、利用者が VNC/noVNC 経由で入力すること
- 背景色と文字色をジョブごとに設定できること

## 6. 表示 URL 要件

### 6.1 URL リスト

- 1 ジョブに複数の表示 URL を登録できること
- 各 URL は以下の属性を持つこと
  - URL
  - priority
  - refresh_interval_sec
- URL の表示順は `priority` 昇順とし、同一 priority は登録順とすること

### 6.2 切替と更新

- ジョブ単位の `refresh_interval_sec` は画面切替間隔として扱うこと
- 複数 URL 設定時は、画面切替間隔ごとに次の URL へ切り替えること
- 各 URL の `refresh_interval_sec` はその URL 自身の再読込間隔として扱うこと
- 各 URL は自身の再読込間隔に従って独立に更新されること
- `refresh_interval_sec=0` の URL は再読込しないこと
- 最小更新間隔は 10 秒とすること

### 6.3 具体例

以下の設定を許容すること。

- 画面切替: 30 秒ごと
- URL1: 1800 秒ごとに再読込
- URL2: 3600 秒ごとに再読込

## 7. 配信要件

- RTMP 配信に対応すること
- SRT 配信に対応すること
- SRT は URL パラメータ付き形式を扱えること
- 配信停止後に再開した場合、新しい設定で再配信されること
- Start / Stop / Restart を WebUI と API から実行できること
- 再接続ポリシーに基づき自動再接続できること

## 8. オーバーレイ要件

- 映像上に日時とメッセージを表示できること
- 表示形式は `YYYY/MM/DD hh:mm:ss {message}` とすること
- 時刻はサーバー時刻を使用すること
- タイムゾーンはコンテナ環境変数 `TZ` に従うこと
- 表示位置は以下から選択できること
  - 左上
  - 右上
  - 左下
  - 右下
- 文字サイズを指定できること
- オーバーレイ表示の有効/無効を切り替えできること

## 9. VNC / noVNC 要件

- Browser 入力および SSH Terminal 入力では VNC 接続を提供できること
- ジョブごとに VNC ポートを割り当てること
- noVNC によりブラウザのみで表示画面へ接続できること
- WebUI の VNC 操作から noVNC URL を開けること
- 利用者は noVNC 上でブラウザ操作や SSH 入力を実施できること

## 10. WebUI 要件

### 10.1 画面構成

- サイドバー付きレイアウトであること
- トップバーにアカウント表示とダークモード切替を持つこと
- 以下の画面を持つこと
  - ジョブ管理
  - ユーザー管理
  - LDAP 設定
  - 監査ログ

### 10.2 ジョブ編集 UI

ジョブ作成・編集画面はタブ構成とすること。

- タブ1: 基本設定
  - 名前
  - 幅
  - 高さ
  - FPS
  - Bitrate
  - Codec
  - Preset
  - Protocol
  - Output URL
  - Stream Key
- タブ2: ソース
  - 入力ソース種別
  - テストパターン設定
  - テストパラメータ
  - SSH Terminal 色設定
  - 画面切替間隔
  - 表示 URL リスト
- タブ3: 追加設定
  - 日時・メッセージ表示有効/無効
  - メッセージ
  - 表示位置
  - 文字サイズ
  - 再接続設定

## 11. API 要件

- API は OpenAPI で定義されていること
- Swagger UI を提供すること
- 認証 API を提供すること
- ジョブ CRUD API を提供すること
- ジョブ開始/停止/再起動 API を提供すること
- ユーザー管理 API を提供すること
- LDAP 設定 API を提供すること
- LDAP 接続確認 API を提供すること
- 監査ログ API を提供すること

## 12. 認証・認可要件

### 12.1 認証方式

- ローカルユーザー認証に対応すること
- LDAP 認証に対応すること
- 初期管理ユーザーは環境変数から投入できること

### 12.2 ロール

以下のロールを提供すること。

- `admin`: 全操作可能
- `operator`: ジョブ運用操作可能
- `viewer`: 参照のみ

### 12.3 LDAP 制約

- LDAP ログイン対象は管理者相当の利用者に限定できること
- OU やグループ条件に合致しない一般ユーザーはログインできないこと
- LDAP 設定には接続確認機能を持つこと

## 13. セキュリティ要件

### 13.1 セッション保護

- セッションは HttpOnly Cookie で保持すること
- Cookie Secure 属性は環境変数で切り替えられること
- セッション期限を設定できること
- ログアウト時はセッションを失効させること

### 13.2 CSRF 対策

- 更新系 API は CSRF トークン検証を必須とすること
- `noc_csrf` Cookie と `x-csrf-token` ヘッダの一致を要求すること

### 13.3 XSS 対策

- CSP を適用すること
- セキュリティヘッダを付与すること
- HSTS の有効/無効を環境変数で切り替えられること
- CSP の `upgrade-insecure-requests` 有効/無効を環境変数で切り替えられること

### 13.4 入力バリデーション

- API 入力は型・範囲・必須性を検証すること
- LDAP 設定入力も妥当性検証を行うこと
- 不正値は保存しないこと

### 13.5 SQL インジェクション対策

- DB アクセスは Prisma を経由して行うこと
- 文字列連結による SQL を避けること

### 13.6 機密情報保護

- `stream_key` を暗号化保存すること
- LDAP `bind_password` を暗号化保存すること
- LDAP `bind_password` は API レスポンスへ返却しないこと
- `JWT_SECRET` と `SECRET_ENCRYPTION_KEY` を必須とすること

## 14. 環境設定要件

- 外部公開ポートを `.env` から指定できること
- MySQL の root/user パスワードを `.env` から指定できること
- `DATABASE_URL` を `.env` から指定できること
- タイムゾーン `TZ` を `.env` から指定できること
- DNS サーバーを `.env` から指定できること
- オフラインモード時は Bootstrap をローカル資産から読み込むこと

## 15. 監査・ログ要件

- 認証、設定変更、ジョブ操作を監査ログに記録すること
- ジョブごとのイベントログを保持すること
- WebUI から監査ログを参照できること
- WebUI からジョブログを参照できること

## 16. 運用要件

- Docker Compose で起動、停止、再構築できること
- テスト環境上で単体動作確認できること
- API 再起動時は不整合な実行状態を補正できること
- オフライン環境での運用を考慮すること
- Linux サーバー上での継続運用を前提とすること
